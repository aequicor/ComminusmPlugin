## Формат ADR

Каждая запись следует следующей структуре:

```
# Решение: Название

## Контекст
...
## Решение
...
## Причины
...
## Последствия
...
```

## Как работать с ADR

1. **Изучите контекст** — поймите проблему, которая привела к решению
2. **Прочитайте причины** — поймите, почему выбрано именно это решение
3. **Оцените последствия** — поймите, как решение повлияет на проект
4. **Обновляйте** — если решение устаревает, создайте новое ADR

## Принципы принятия решений

- Все решения документируются
- Аргументированное обоснование
- Оценка последствий
- Обновление при изменении контекста

---

**Примечание**: АDR являются частью documents и должны обновляться при изменении архитектуры.
