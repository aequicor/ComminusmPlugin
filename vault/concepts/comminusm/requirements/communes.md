**Status:** Draft

## Business Context

Игроки объединяются в команды через систему ордеров. Ордер — автономная единица: свой флаг, лидер, территория (приваты и трудовые фронты), точка спавна и внутренний чат. Однако между несколькими ордерами не существует механизма постоянного союза: нет общего канала связи, нет защиты от взаимного урона, нет возможности дать союзному ордеру права изменять территорию.

**Коммуна** — надстройка над ордерами, которая решает три конкретные проблемы:

1. **Совместное управление территорией** — игрок из союзного ордера может получить права member'а в другом ордере коммуны, что позволяет ему изменять приваты и трудовые фронты этого ордера. Механика переиспользует уже существующую роль Member ордера — новый игровой примитив не вводится.

2. **Общение внутри союза** — все участники ордеров, входящих в коммуну, получают единый чат коммунаров, отделённый от общего и внутриордерного чатов.

3. **Дружественный огонь** — члены одной коммуны не наносят урон друг другу (friendly-fire отключён для коммунаров).

Коммуна строится на принципе open-closed (SOLID): существующий код ордеров не модифицируется. Функциональность добавляется исключительно через новые компоненты.

Каждый ордер полностью сохраняет свою идентичность: флаг, лидер, территория, /home, состав member'ов остаются ордерными. Коммуна не является заменой ордера — это альянс ордеров.

## User Stories

| # | As a... | I want to... | So that... |
|---|---------|-------------|-----------|
| US-01 | Лидер ордера | Создать коммуну | Я могу объединить свой ордер с другими ордерами в союз |
| US-02 | Лидер ордера | Пригласить другой ордер в коммуну | Союзный ордер может присоединиться к нашей коммуне |
| US-03 | Лидер ордера (получивший приглашение) | Принять или отклонить приглашение в коммуну | Я управляю тем, в каких союзах участвует мой ордер |
| US-04 | Лидер ордера | Выйти из коммуны | Мой ордер снова становится полностью независимым |
| US-05 | Лидер ордера | Открыть меню коммуны | Я могу видеть состав коммуны и управлять им |
| US-06 | Любой коммунар | Открыть меню коммуны через кнопку в меню партии | Я быстро получаю доступ к управлению коммуной без поиска команды |
| US-07 | Лидер ордера | Добавить игрока из союзного ордера как member'а в свой ордер через меню коммуны | Союзный игрок получает права на изменение моих приватов и трудовых фронтов |
| US-08 | Лидер ордера | Удалить игрока из союзного ордера из списка member'ов своего ордера | Права cross-order доступа отзываются |
| US-09 | Любой коммунар | Писать в чат коммуны | Только члены ордеров коммуны получат сообщение |
| US-10 | Любой коммунар | Не получать урон от участников своей коммуны | Я могу сражаться рядом с союзниками без риска случайного урона |
| US-11 | Любой игрок | Видеть, к какой коммуне принадлежит чужой ордер | Я понимаю структуру союзов на сервере |

## Acceptance Criteria

| ID | Story | Given | When | Then |

| AC-06 | US-02 | Лидер ордера пытается пригласить ордер, который уже состоит в той же коммуне | Отправка приглашения | Приглашение отклоняется с уведомлением: «Указанный ордер уже в вашей коммуне» |
| AC-07 | US-03 | Лидер ордера получил приглашение в коммуну | Лидер принимает приглашение | Ордер добавляется в коммуну; все действующие члены коммуны получают уведомление о новом ордере |
| AC-08 | US-03 | Лидер ордера получил приглашение в коммуну | Лидер отклоняет приглашение | Ордер не добавляется в коммуну; приглашение удаляется |
| AC-09 | US-03 | Приглашение в коммуну не принято и не отклонено | Истекает таймаут приглашения | Приглашение автоматически истекает, ордер не вступает в коммуну |
| AC-10 | US-04 | Лидер ордера, ордер состоит в коммуне с двумя и более ордерами | Лидер выходит из коммуны | Ордер исключается из коммуны; cross-order member-права, выданные игрокам этого ордера в других ордерах коммуны, отзываются; остальные члены коммуны получают уведомление |
| AC-11 | US-04 | Лидер единственного оставшегося ордера в коммуне выходит | Выход последнего ордера | Коммуна расформировывается |
| AC-12 | US-05 | Любой коммунар | Игрок открывает меню коммуны | В меню отображаются: название коммуны, список ордеров-участников, кнопки управления (приглашение, выход) |
| AC-13 | US-06 | Любой коммунар находится в меню партии | Игрок нажимает кнопку «Коммуна» | Открывается меню коммуны |
| AC-14 | US-06 | Игрок не состоит ни в одной коммуне | Игрок открывает меню партии | Кнопка «Коммуна» видна, но недоступна (disabled) или открывает экран создания коммуны |
| AC-15 | US-07 | Лидер ордера A, ордер A состоит в коммуне с ордером B | Лидер A добавляет игрока из ордера B как member'а ордера A через меню коммуны | Игрок из ордера B получает роль Member ордера A; он может изменять приваты и трудовые фронты ордера A |
| AC-16 | US-07 | Лидер ордера A пытается добавить как member'а игрока, не принадлежащего ни к одному ордеру коммуны | Добавление через меню коммуны | Добавление отклоняется с уведомлением: «Игрок не является членом ордеров этой коммуны» |
| AC-17 | US-08 | Лидер ордера A, игрок из ордера B является member'ом ордера A | Лидер A удаляет cross-order member'а через меню коммуны | Игрок из ордера B теряет роль Member ордера A; права на изменение территории ордера A отзываются |
| AC-18 | US-09 | Любой коммунар | Игрок пишет в чат коммуны (через отдельную команду или переключение режима чата) | Сообщение доставляется всем коммунарам и только им; сообщение отображается с пометкой [Коммуна] |
| AC-19 | US-09 | Игрок не является коммунаром | Игрок пытается написать в чат коммуны | Сообщение не принимается; игрок получает уведомление об отсутствии коммуны |
| AC-20 | US-10 | Два игрока принадлежат к ордерам одной коммуны | Один из игроков атакует другого (любой тип атаки: ближний бой, дальний, взрывы, статусные эффекты через проджектайл) | Урон не наносится; атакующий игрок получает уведомление «Это ваш союзник» |
| AC-21 | US-10 | Два игрока принадлежат к ордерам разных коммун (или один без коммуны) | Один из игроков атакует другого | PvP работает в штатном режиме |
| AC-22 | US-10 | Игрок является member'ом ордера A через cross-order доступ, а его «родной» ордер и ордер A — в одной коммуне | Игрок атакует другого коммунара | Урон не наносится (friendly-fire определяется по принадлежности «родного» ордера к коммуне, не по cross-order member-статусу) |
| AC-23 | US-11 | Любой игрок | Игрок смотрит информацию об ордере (через меню или команду) | Отображается название коммуны, в которой состоит ордер (если состоит) |

## Out of Scope

- Коммуна не имеет собственной территории — она не создаёт и не управляет приватами или трудовыми фронтами.
- Коммуна не имеет собственного флага или точки спавна.
- Ограничение числа ордеров в коммуне — ограничений нет.
- Ограничение числа участников коммуны — ограничений нет.
- Иерархия ролей внутри коммуны (кроме «лидер ордера имеет управленческие права в коммуне») — не предусмотрена.
- Передача лидерства коммуны как отдельной сущности — лидером коммуны является лидер ордера-основателя (или лидер ордера, который последним управлял коммуной; точная политика — открытый вопрос).
- Глобальный рейтинг или статистика коммун — вне scope данной функции.
- Коммунальный склад (shared inventory) — не входит в данную задачу.
- Общий respawn у флага коммуны — коммуна не имеет флага; respawn остаётся ордерным.
- Изменение чужого ордерного чата участниками коммуны — чат ордера остаётся ордерным; добавляется только отдельный коммунальный чат.
- Исключение отдельного игрока из коммуны (без выхода всего ордера) — управление составом происходит на уровне ордеров, не на уровне отдельных игроков.
- Модификация существующего кода ордеров — функциональность добавляется исключительно через новые компоненты (open-closed).
- Административные команды для принудительного управления коммунами — вне scope базовой реализации.

## Dependencies

- **Система ордеров и роль Member** — cross-order доступ к территории реализуется через уже существующую роль Member ордера. Система ордеров должна поддерживать добавление member'ов извне (из другого ордера).
- **flag-stability** — флаги ордеров идентифицируют территорию; на их базе работают приваты и трудовые фронты, к которым cross-order member получает доступ.
- **privates-orders-fronts** — механика изменения приватов и трудовых фронтов, к которой cross-order member'ы получают доступ.
- **PartyMenu / OrderMenu** — UX-паттерн для меню коммуны. Кнопка «Коммуна» интегрируется в существующее меню партии.
- **Система чата** — добавляется новый канал чата коммунаров; должна существовать точка расширения или команда переключения чата.
- **Bukkit/Paper EntityDamageByEntityEvent** — для реализации friendly-fire защиты.
