#如何 создать документацию

## Предварительные условия

- Доступ к репозиторию ComminusmPlugin
- Знание Diátaxis фреймворка
- Понимание темы, которую нужно задокументировать

## Шаг 1: Определите жанр

Выберите правильный жанр для документа:

- **concepts/** — объяснение архитектуры, принятых решений, фундаментальных принципов
- **guidelines/** — правила, которым нужно следовать при разработке
- **how-to/** — пошаговые инструкции для выполнения задач
- **reference/** — факты, API, конфигурации

## Шаг 2: Выберите тему

Используйте kebab-case имя для темы (например: `architecture`, `search-rule`, `add-feature`).

## Шаг 3: Создайте файл

Создайте новый Markdown файл в соответствующей директории:

- `vault/concepts/имя.md` — для концепций
- `vault/concepts/decisions/NNN-имя.md` — для ADR
- `vault/guidelines/имя.md` — для правил
- `vault/how-to/имя.md` — для инструкций
- `vault/reference/имя.md` — для справочников

## Шаг 4: Добавьте Frontmatter

Каждый документ должен включать frontmatter:

```yaml
---
genre: concept | guideline | how-to | reference
title: Человекочитаемый заголовок на русском
topic: kebab-case-тема
confidence: high | medium | low
source: human | agent
updated: YYYY-MM
---
```

## Шаг 5: Напишите контент

### Для concepts:
- Объясняйте "что" и "почему"
- Для ADR: контекст → решение → причины → последствия

### Для guidelines:
- Одно правило = одна тема
- Задайте конкретные триггеры для поиска
- Не добавляйте объяснения — только правила

### Для how-to:
- Начните с предварительных условий
- Следующие шаги пронумеруйте
- Завершите шагами проверки

### Для reference:
- Только факты, без мнений
- Кратко и понятно
- Удобно для копирования и вставки

## Шаг 6: Обновите индекс

Добавьте вики-ссылку на новый документ в `vault/_INDEX.md`:

```markdown
- [[guidelines/имя|Заголовок документа]]
```

## Шаг 7: Добавьте взаимные ссылки

Добавьте `related: ["[[путь/к/связанному]]"]` в frontmatter нового документа.

## Шаг 8: Проверьте доступность

Выполните поиск через `knowledgeos_search_docs` с темой документа, чтобы убедиться, что он находится.

---

## Пример минимального документа

```markdown
---
genre: guideline
title: Пример правила
triggers: ["пример", "test"]
tags: ["process"]
related: []
---

# Пример правила

Короткое описание правила.

## Обязательное содержимое

Здесь пишутся конкретные правила, которым нужно следовать.
```

---

## Валидация

После создания документа:

1. ✅ Проверьте структуру директорий
2. ✅ Проверьте наличие frontmatter
3. ✅ Убедитесь, что заголовок на русском
4. ✅ Проверьте файл в git (не забыли ли добавить?)
5. ✅ Выполните локальный поиск по документу

---

**Примечание**: Документация — единственный источник истины. Если что-то не документировано, это считается отсутствующим знанием.
