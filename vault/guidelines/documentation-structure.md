# Структура документации

Вся документация проекта ComminusmPlugin должна следовать фреймворку [Diátaxis](https://diataxis.fr/), организованному в четыре жанра.

## Структура директорий

```
vault/
├── _INDEX.md                    # Главный индекс с вики-ссылками
├── concepts/                    # Концепции (объяснения, ADR)
│   ├── architecture.md          # Обзор архитектуры
│   └── decisions/               # Записи архитектурных решений
│       ├── 001-name.md          # Формат: NNN-описательное-имя
│       └── 002-name.md
├── guidelines/                  # Правила (действия)
│   └── name-of-rule.md
├── how-to/                      # Пошаговые инструкции
│   └── action-to-perform.md
└── reference/                   # Факты, API, конфигурация
    └── topic-of-reference.md
```

## Жанр: concepts/

Содержит:
- Объяснения архитектуры, принятые решения, фундаментальные принципы
- ADR-файлы: нумерация (`001-`, `002-`...), kebab-case после номера
- Frontmatter: `genre: concept`, `title`, `topic`, `confidence`, `source`, `updated`
- ADR структура: Контекст → Решение → Причины → Последствия
- Нет пошаговых инструкций (для how-to/)

## Жанр: guidelines/

Содержит:
- Правила, которым должен следовать AI-агент
- Frontmatter: `genre: guideline`, `triggers` (обязательно), `tags`, `related`
- Один гайдлайн = одна тема
- Указание конкретных триггеров для поиска
- Нет объяснений (контекст в concepts/)

## Жанр: how-to/

Содержит:
- Пошаговые инструкции, гайды по внедрению
- Frontmatter: `genre: how-to`, упорядоченный `steps` массив
- Имя файла: ориентировано на действие (`add-mcp-tool.md`, не `mcp-tool-creation`)
- Начинается с предварительных условий, заканчивается шагами проверки
- Нет концептуальных объяснений (в concepts/)

## Жанр: reference/

Содержит:
- Факты, API, конфигурацию, структуры данных
- Frontmatter: `genre: reference`, `version`
- Нет мнений и объяснений — только факты
- Минимальный объем, удобный для копирования

## Именование

1. **Файлы:** kebab-case, описательные, расширение `.md`
2. **Вики-ссылки:** совпадают с именами файлов: `[[concepts/architecture]]`
3. **Индекс:** всегда использовать `[[путь/к/файлу]]` в `_INDEX.md`
4. **ADR:** последовательная нумерация с описательными суффиксами

## Необходимый frontmatter

Каждый документ ДОЛЖЕН включать:

```yaml
---
genre: concept | guideline | how-to | reference
title: Человекочитаемый заголовок на русском
topic: kebab-case-тема
confidence: high | medium | low
source: human | agent
updated: ГГГГ-ММ
---
```

## Рабочий процесс

1. Новое знание → создать в соответствующем жанре
2. Обновить `_INDEX.md` вики-ссылкой на новый документ
3. Добавить вики-ссылки на взаимосвязанные документы в обоих направлениях
4. Выполнить поиск для проверки доступности через триггеры/тему
