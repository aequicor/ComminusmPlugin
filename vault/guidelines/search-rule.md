# Правило поиска

При поиске информации всегда следуйте этому приоритету:

## Приоритет 1: Документация (KnowledgeOS)

Всегда используйте `knowledgeos_search_docs` для любого вопроса о документации проекта, концепциях, правилах, архитектуре, решениях или конвенциях. Vault — единственный источник истины.

## Приоритет 2: Чтение результата

`knowledgeos_search_docs` возвращает `chunkIds` с `docPath`. Прочтите каждый уникальный `docPath` из `vault/`, чтобы получить полный контент, прежде чем решать, нужна ли дополнительная информация.

## Приоритет 3: Поиск в коде

Только если `knowledgeos_search_docs` не дал полезного результата И чтение документов не помогло, используйте `grep`, `glob` или `read` для поиска в кодовой базе.

## Приоритет 4: Документирование

Если вы обнаружили что-то важное, чего нет в vault, используйте `knowledgeos_write_guideline`, чтобы задокументировать это.

## Формат запроса

1. Определите тему поиска
2. Выполните `knowledgeos_search_docs` с query (`topic: ...`, `genre: ...`)
3. Прочитайте все возвращенные документы
4. Если ответ не найден — ищите код
5. Если нашли новую важную информацию — документируйте её
