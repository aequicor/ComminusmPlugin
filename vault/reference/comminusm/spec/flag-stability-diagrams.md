### Sequence — US-01/US-02: Flag Activation (Order / Front)

```mermaid
    FlagListener->>FlagListener: check world in allowedWorlds
    FlagListener->>Cache: check position occupied (cold-start: fallback to PDC)
    FlagListener->>FlagListener: check maxPerChunk
    FlagListener->>FlagListener: check minAirAbove
    FlagListener->>Manager: tryLock(chunkKey) — non-blocking
    alt lock acquired
        Manager->>FlagListener: lock granted
        FlagListener->>FlagListener: re-check position occupied (double-check)
        FlagListener->>FlagListener: capture originalMaterial (local val)
        FlagListener->>PDC: write comminusm:flag/{id}
        FlagListener->>World: replace support block → flagSupportBlockMaterial
        FlagListener->>PDC: write comminusm:support_material/{id}
        FlagListener->>Cache: add banner + support positions to flagPositionCache
        FlagListener->>Manager: unlock(chunkKey)
        FlagListener->>Scheduler: runTaskAsynchronously (DB save)
        Scheduler-->>DB: save order/front record
        alt DB save success
            DB-->>Scheduler: ok
            Scheduler->>Scheduler: runTask (main thread callback)
            Scheduler->>Manager: tryLock re-acquire (max 5 retries)
            Manager->>PDC: verify comminusm:flag/{id} still present
            Manager->>World: spawn ArmorStand at bannerY+2
            Manager->>PDC: write comminusm:armorstand/{id}
            Manager->>Manager: unlock
            Manager->>Player: send success message (if online)
        else DB save failure
            DB-->>Scheduler: error
            Scheduler->>Scheduler: runTask (Rollback Step R)
            note over Scheduler: Rollback: remove ArmorStand,\ndelete PDC keys, restore support block\nto originalMaterial
            Scheduler->>Player: send error message (if online)
        end
    else lock not acquired
        Manager->>Player: "Попробуйте ещё раз через секунду"
    end
```

### Sequence — ChunkLoadEvent: Crash-Recovery Repair

```mermaid
sequenceDiagram
    participant Bukkit
    participant ChunkLoadListener
    participant Manager as FlagStabilityManager
    participant PDC as Chunk PDC
    participant World
    participant Scheduler as BukkitScheduler
    participant DB

    Bukkit->>ChunkLoadListener: ChunkLoadEvent(chunk)
    ChunkLoadListener->>Manager: onChunkLoad(chunk)

    loop For each comminusm:flag/{id} key in chunk PDC
        Manager->>PDC: check comminusm:dirty_armorstand/{id} (Step A)
        alt dirty_armorstand key present
            Manager->>World: find ArmorStand by UUID
            opt entity found
                Manager->>World: entity.remove()
            end
            Manager->>PDC: delete comminusm:dirty_armorstand/{id}
        end

        Manager->>PDC: resolve banner pos from comminusm:flag/{id}

        Manager->>PDC: check comminusm:support_material/{id} (Step C)
        alt support_material ABSENT
            Manager->>World: check actual support block material
            alt material is BEDROCK or OBSIDIAN
                note over Manager: new flag, crashed before writing support_material\n→ proceed to Step E (repair)
            else other material
                note over Manager: legacy flag → skip, continue
            end
        else support_material PRESENT and armorstand ABSENT
            note over Manager: partial PDC write (Q5)\n→ proceed to Step E Case 1
        else support_material PRESENT and armorstand PRESENT
            Manager->>PDC: read comminusm:support_material/{id} (Step D passive verify)
            Manager->>World: compare actual block vs PDC material
            alt materials match
                note over Manager: go to Step F (name refresh)
            else actual block is AIR
                note over Manager: go to Step E (lazy repair / deletion disambiguation)
            else actual block different non-AIR material
                Manager->>World: restore support block to PDC material
            end
        end
    end

    note over Manager,Scheduler: Flags needing DB consultation collected (Step E candidates)
    Manager->>Scheduler: runTaskAsynchronously (batch DB queries for all E-candidates)
    Scheduler-->>DB: query each flag id — active?
    DB-->>Scheduler: results list
    Scheduler->>Scheduler: runTask (main thread callback)

    loop For each E-candidate result
        alt DB ERROR
            note over Manager: fail-safe: log WARN, skip, no world changes
        else armorstand key ABSENT and DB PRESENT (Case 1 — activation repair)
            Manager->>World: create ArmorStand, update PDC comminusm:armorstand/{id}
        else armorstand key PRESENT, entity not found, DB PRESENT (Case 2 — orphan repair)
            alt support block is AIR
                Manager->>World: restore support block to PDC support_material
            end
            Manager->>World: recreate ArmorStand, update PDC
        else armorstand key PRESENT, entity not found, DB ABSENT (Case 2 — deletion cleanup)
            Manager->>World: replace support block → AIR
            Manager->>World: delete banner block → AIR
            Manager->>PDC: delete all comminusm:*/{id} keys
        end
    end

    Manager->>Manager: rebuild flagPositionCache for this chunk (Step F)
    loop For each flag with armorstand
        Manager->>World: find ArmorStand by UUID
        opt entity found and name differs
            Manager->>World: update ArmorStand customName (lazy name refresh)
        end
    end
```

### Sequence — Startup Repair Scan

```mermaid
sequenceDiagram
    participant ComminusmPlugin
    participant Scheduler as BukkitScheduler
    participant DB
    participant World
    participant PDC as Chunk PDC
    participant Cache as flagPositionCache

    ComminusmPlugin->>Scheduler: onEnable → runTaskAsynchronously (Phase 1)
    Scheduler-->>DB: query all active order + front flags (id, bannerPos, worldName)
    DB-->>Scheduler: full record list
    Scheduler->>Scheduler: runTask (main thread — Phase 2 start)

    alt total flags > 100
        Scheduler->>Scheduler: log WARN (count + batch size)
    end

    loop Each batch of flagStartupScanBatchSize flags (tick-spread)
        loop Each DB record in batch
            Scheduler->>World: world.isChunkLoaded(cx, cz) — safe on main thread
            alt chunk not loaded
                note over Scheduler: skip — ChunkLoadEvent handles on next access
            else chunk loaded
                Scheduler->>PDC: check comminusm:flag/{id} exists
                alt PDC key absent
                    Scheduler->>PDC: write comminusm:flag/{id}
                    Scheduler->>PDC: write comminusm:support_material/{id}
                    Scheduler->>PDC: check comminusm:armorstand/{id} exists
                    alt armorstand key present
                        Scheduler->>World: resolve ArmorStand by UUID
                        alt entity found
                            note over Scheduler: skip spawn (already exists)
                        else entity not found (stale UUID)
                            Scheduler->>World: spawn new ArmorStand
                            Scheduler->>PDC: overwrite comminusm:armorstand/{id}
                        end
                    else armorstand key absent
                        Scheduler->>World: spawn ArmorStand
                        Scheduler->>PDC: write comminusm:armorstand/{id}
                    end
                    Scheduler->>Cache: add positions to flagPositionCache
                end
            end
        end
        Scheduler->>Scheduler: schedule next batch via runTask (1-tick delay)
    end

    Scheduler->>Scheduler: log INFO "Startup repair scan complete. Repaired: N flags."
```

## State — Flag Lifecycle

```mermaid
stateDiagram-v2
    [*] --> INACTIVE : flag item in player inventory

    INACTIVE --> ACTIVATING : player places banner\n(pre-conditions pass, lock acquired)

    ACTIVATING --> ACTIVE : ArmorStand spawned,\nPDC keys written, DB saved

    ACTIVATING --> ROLLBACK_PENDING : DB save fails OR\nArmorStand spawn fails

    ROLLBACK_PENDING --> INACTIVE : rollback complete\n(support block restored,\nPDC keys deleted,\nplayer message sent)

    ROLLBACK_PENDING --> DIRTY_ARMORSTAND : ArmorStand.remove() fails\nduring rollback

    DIRTY_ARMORSTAND --> INACTIVE : ChunkLoadEvent cleans\ndirty_armorstand marker

    ACTIVE --> MOVE_IN_PROGRESS : owner moves front\n(Step 3 DB update begins)

    MOVE_IN_PROGRESS --> ACTIVE : move completed\n(new ArmorStand + PDC + support block\nplaced at new position)

    MOVE_IN_PROGRESS --> ACTIVE : Step 13R rollback\n(new pos rolled back;\nstartup scan re-completes\non next server start)

    ACTIVE --> DEACTIVATING : deleteByOwner or\ndeactivate called,\nchunk lock acquired

    DEACTIVATING --> DELETED : world cleanup complete\n(support block AIR,\nArmorStand removed,\nPDC keys deleted,\nDB record deleted)

    ACTIVE --> CRASHED_PARTIAL : server crash mid-activation\nor mid-deletion

    CRASHED_PARTIAL --> ACTIVE : ChunkLoadEvent repair\n(Section 9 Step E)\nor Startup Repair Scan

    CRASHED_PARTIAL --> DELETED : ChunkLoadEvent repair\ndetects DB absent →\nworld cleanup completed

    DELETED --> [*]
```
